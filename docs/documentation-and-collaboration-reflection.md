# Reflection: Documentation & Collaboration

### Original Assumptions
At first, our README assumed that anyone reading it already knew everything about our project. We expected them to know the tech stack, where files were located, and how to set up the environment without us having to write it down.

### Onboarding Improvements
The new README fixes this by acting as a "single source of truth". By clearly listing prerequisites, simplifying the installation steps, and explaining the project structure, we’ve made it possible for a new developer to get the system running on their own on their first day.

### What I Learned
I learned that Pull Requests are a great way to communicate with the team, not just a way to merge code. By writing a clear description, I can explain why I made my changes and let reviewers know what to look for. This helps avoid confusion and keeps the review process moving quickly without any unnecessary delays.
